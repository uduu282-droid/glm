## 🔐 WHAT PIXELBIN REQUIRES

### Required Authentication Headers:

```javascript
Headers Needed:
1. pixb-cl-id       → Dynamic client ID (changes per session)
2. x-ebg-param      → Timestamp parameter (changes every request)
3. x-ebg-signature  → Cryptographic signature (changes every request)
4. input.captchaToken → Captcha solution token (EXPIRES IN 5-10 MIN!)
```

### Where These Come From:

| Token | Source | Lifetime |
|-------|--------|----------|
| `pixb-cl-id` | Browser session cookies | Until browser closes |
| `x-ebg-param` | JavaScript on page load | Single use |
| `x-ebg-signature` | Generated by page JS | Single use |
| `captchaToken` | **Solving captcha on website** | 5-10 minutes ⚠️ |

## 💡 THE CAPTCHA PROBLEM

### What is Captcha?

**CAPTCHA** = Completely Automated Public Turing test to tell Computers and Humans Apart

**Purpose:** Prove you're a real human, not a bot/script

### Pixelbin's Captcha:

1. Website shows captcha challenge (invisible or visible)
2. User solves it (click images, checkbox, etc.)
3. Captcha service returns a token
4. Token is included in API request
5. Server validates token before processing

**Without valid captcha token → 403 Forbidden**

## 🎯 BOTTOM LINE

### Why Browser Automation?

1. **Captcha Required** - Can't bypass it, must solve it
2. **Dynamic Tokens** - Change every request/session
3. **Token Expiration** - Die quickly, need constant refresh
4. **Easiest Solution** - Let human solve on website, capture result

### Is There a Better Way?

**For free users?** No - browser automation is the simplest reliable method

**For paid users?** Could use official API (if they offer one)
