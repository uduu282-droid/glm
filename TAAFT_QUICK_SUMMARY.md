### ❌ What's Blocking Direct API Access

The `se_accept_encoding` header is generated by client-side JavaScript that:
- Intercepts XMLHttpRequest calls
- Generates random strings using Base62 alphabet
- Inserts random characters into request paths
- Uses localStorage tokens (`candy_shoop`, `refParam`)

**Result**: Direct API calls return `403 Forbidden` or empty `200 OK` responses

## 🔍 Key Findings

### How the Security Works

```javascript
// From page source (line ~849)
XMLHttpRequest.prototype.open = function(method, url, ...) {
    // Check domain
    if (/^theresanaiforthat\.com$/.test(hostname)) {
        // Generate random strings
        var random4 = generateRandom(4);
        var random17 = generateRandom(17);
        var random20 = generateRandom(20);
        
        // Obfuscate path
        var obfuscatedPath = url.slice(0,4) + random17 + url.slice(4);
        
        // Create header
        se_accept_encoding = random4 + hostname + obfuscatedPath + random20;
        
        this.setRequestHeader("se_accept_encoding", se_accept_encoding);
    }
};
```
